## 2. 核心功能模块
### 2.1 用户认证模块
**涉及文件**：

```plain
web/src/components/Login.js
web/src/services/api.js
server/routes/auth.js
server/middleware/auth.js
server/db/index.js
server/config.js
```

**运作逻辑**：

1. 前端 `Login.js` 收集用户名/密码，调用 `api.post('/auth/login')`。
2. 后端 `auth.js` 的 `POST /login`：
    - 通过 `db.getUserByUsername()` 查询 `users` 表。
    - 使用 `bcrypt.compare()` 验证密码哈希。
    - 生成 JWT（有效期 1 小时，密钥来自 `JWT_SECRET` 环境变量或默认值）。
    - 返回 `{ success: true, token, user: { username, role } }`。
3. 前端将 token 存入 `localStorage.setItem('auth_token', token)`，跳转首页。
4. `api.js` 请求拦截器自动添加 `Authorization: Bearer <token>` 头。
5. 后端 `auth.js` 中间件验证 JWT，解析用户信息挂载到 `req.user`。
6. 支持刷新令牌：`POST /auth/refresh-token`（允许过期 token 刷新）。
7. 登出仅前端移除 token，无后端状态。

**数据库表结构** (`users`)：

| 字段 | 类型 | 说明 |
| :--- | :--- | :--- |
| `id` | INTEGER PK | 自增 |
| `username` | TEXT UNIQUE | 用户名 |
| `passwordHash` | TEXT | bcrypt 哈希 |
| `role` | TEXT | 角色（admin/user） |
| `createdAt` | DATETIME | 创建时间 |


**FISCO BCOS 迁移影响**：**无**。完全独立于区块链。

### 2.3 图片上传与哈希计算模块
**涉及文件**：

```plain
web/src/components/ImageUpload.js
server/routes/hash.js
server/utils/hashCalculator.js
server/config.js
web/src/services/api.js
```

**运作逻辑**：

1. 前端 `ImageUpload.js` 选择文件，调用 `api.calculateHash(file)`。
2. 后端 `hash.js` 的 `POST /calculate-hash`：
    - 使用 `multer` 内存存储接收文件 Buffer。
    - 调用 `hashCalculator.calculateImageHash(buffer)`。
    - 返回 `{ success: true, hash }`（`0x` + 64 位十六进制）。
3. 哈希计算：`crypto.createHash('sha256').update(buffer).digest('hex')`，与区块链无关。
4. 前端显示哈希，供后续上链使用。

**FISCO BCOS 迁移影响**：**无**。SHA-256 为标准算法。



### 2.4 哈希上链模块
**涉及文件**：

```plain
web/src/components/ImageUpload.js
server/routes/hash.js
scripts/uploadImage.js
scripts/config.js
contracts/MyToken.sol (ABI)
artifacts/contracts/MyToken.sol/MyToken.json
```

**运作逻辑**：

1. 前端点击“上传到区块链”，调用 `api.uploadHash(formData)`，包含 `hash` 和图片文件。
2. 后端 `hash.js` 的 `POST /upload-to-blockchain`：
    - 通过 `execAsync` 执行命令：`node scripts/uploadImage.js "<hash>"`。
    - 从 stdout 中正则提取 `blockNumber: (\d+)`。
    - 将图片文件保存到 `IMAGE_STORAGE_PATH/{hash}.jpg`。
    - 调用 `db.addImageRecord(hash, imagePath, blockNumber)` 写入数据库。
3. 脚本 `uploadImage.js`：
    - 使用 `ethers` 连接 RPC (`http://localhost:8545`)。
    - 创建 `Wallet` 签名者。
    - 调用合约 `isImageHashActive.staticCall(hash)` 检查存在性。
    - 若不存在，发送 `uploadImageHash(hash)` 交易并等待收据。
    - 输出 `blockNumber: xxx` 和 `transactionHash: xxx`。
4. 合约 `MyToken.sol`：
    - `mapping(bytes32 => bool) activeHashes` 记录活跃状态。
    - `bytes32[] allImageHashes` 记录所有历史哈希。
    - 触发 `ImageHashUploaded` 事件。

**关键路径**：

```plain
ImageUpload.js → api.uploadHash → 后端 /upload-to-blockchain → exec(scripts/uploadImage.js) → ethers → 合约
```

**FISCO BCOS 迁移影响**：**中高**。需将 `uploadImage.js` 中的 ethers 替换为 FISCO BCOS SDK 的对应方法（`Web3`、`Contract`、`.send()`）。



### 2.5 哈希验证模块
**涉及文件**：

```plain
web/src/components/HashVerification.js
server/routes/hash.js
scripts/verifyWithHistory.js
```

**运作逻辑**：

1. 前端支持两种模式：
    - **文件模式**：上传图片 → 计算哈希 → 验证。
    - **输入模式**：直接输入 66 字符哈希（`0x` 开头）。
2. 调用 `api.verifyHash(hash)`。
3. 后端 `hash.js` 的 `POST /verify-hash`（需认证）：
    - 通过 `runScript('verifyWithHistory.js', [hash])` 执行外部脚本。
    - 检查 stdout 是否包含 `✅ 哈希已存在于链上`。
    - 返回 `{ success: true, isVerified, message }`。
4. 脚本 `verifyWithHistory.js`：
    - 连接 RPC，调用合约 `isImageHashActive(hash)` 查询。
    - 输出结果字符串。

**备选接口**：`POST /check-existence` 直接使用 ethers 查询，但前端未使用。

**FISCO BCOS 迁移影响**：**中**。需重写 `verifyWithHistory.js` 为 FISCO BCOS SDK 查询。

### 3.3 批量操作工具
**涉及文件**：

```plain

scripts/batchVerifyWithHistory.js
scripts/blockchain-uploader.js
```

**运作逻辑**：

+ `batchUpload.js`：命令行工具，扫描目录图片，计算 SHA-256 哈希，批量上链，生成报告，移动成功文件。内置 nonce 管理器。

**FISCO BCOS 迁移影响**：**中**。需替换 SDK 调用，并修正 `batchVerifyWithHistory.js` 的哈希算法为 SHA-256。
